# 管理后台使用手册

## 1. 登录

### 1.1 首次登录

1. 访问管理后台地址（如 `http://admin.parking.example.com`）
2. 使用系统初始化时生成的用户名和密码登录
3. 首次登录强制修改密码
4. 密码要求：至少 8 位，包含大小写字母、数字和特殊字符

### 1.2 日常登录

1. 输入用户名和密码
2. 点击「登录」

**注意：** 连续 5 次登录失败将锁定账号，需联系 Super_Admin 解锁。

## 2. 业主审核

### 2.1 单个审核

1. 进入「业主审核」页面
2. 查看待审核业主列表（手机号、房屋号、身份证后 4 位）
3. 点击「通过」或「驳回」
4. 驳回时需填写驳回原因
5. 审核结果通过订阅消息通知业主

### 2.2 批量审核

1. 在列表中勾选多个待审核记录（最多 50 条）
2. 点击「批量通过」或「批量驳回」
3. 确认操作

## 3. 车辆管理

1. 进入「车辆管理」页面
2. 支持按小区、房屋号、车牌号筛选
3. 查看车辆状态（`normal` / `primary`）
4. 查看车辆详细信息

## 4. Visitor 审批

### 4.1 单个审批

1. 进入「Visitor 审批」页面
2. 查看待审批申请（业主信息、车牌号、申请原因）
3. 点击「通过」或「驳回」
4. 驳回时需填写驳回原因

### 4.2 批量审批

1. 勾选多个待审批记录（最多 50 条）
2. 点击「批量通过」或「批量驳回」
3. 确认操作

## 5. 车位配置

1. 进入「车位配置」页面
2. 查看当前配置：总车位数、预留车位数、月度配额、单次时长限制
3. 点击「修改配置」
4. 填写新的配置值
5. 点击「保存」

**约束：** 新的总车位数不能小于当前在场车辆数。

## 6. 报表查看

进入「报表」页面，可查看以下报表：

### 6.1 入场趋势

- 折线图展示入场车辆数量变化
- 支持按日、周、月聚合
- 支持自定义时间范围

### 6.2 车位使用率

- 柱状图展示车位使用率
- 计算公式：平均在场车辆数 / 总车位数

### 6.3 峰值时段

- 热力图展示高峰时段分布
- 帮助优化车位管理策略

### 6.4 僵尸车辆统计

- 展示僵尸车辆数量和处理情况

所有报表支持导出数据。

## 7. 僵尸车辆处理

1. 进入「僵尸车辆」页面
2. 查看僵尸车辆列表（车牌号、入场时间、连续在场天数）
3. 支持按状态筛选（未处理 / 已联系 / 已解决 / 已忽略）
4. 点击「处理」，选择处理方式：
   - `contacted`：已联系车主
   - `resolved`：已解决
   - `ignored`：忽略
5. 填写处理记录

**僵尸车辆定义：** 连续在场超过 7 天的车辆，系统每日自动识别。

## 8. 审计日志

1. 进入「审计日志」页面
2. 支持按操作类型、操作人、时间范围筛选
3. 查看操作日志详情（包含操作前后的状态变化）
4. 支持导出审计日志（需 Super_Admin 权限）

**说明：** 默认显示最近 30 天日志，6 个月以上的日志自动归档。

## 9. 常见问题

**Q: 账号被锁定怎么办？**
A: 联系 Super_Admin 解锁账号。

**Q: 修改车位总数后 Visitor 名额变化了？**
A: 修改总车位数会重新计算 Visitor_Available_Spaces，可能导致部分已批准的 Visitor 授权变为不可用。

**Q: 批量操作部分失败怎么办？**
A: 系统会返回成功和失败的详细信息，状态不正确的记录会被跳过并标注原因。

**Q: 如何查看异常出场记录？**
A: 异常出场记录会自动通知物业管理员，可在入场记录中按状态筛选 `exit_exception` 类型。

**Q: 如何导出原始数据（未脱敏）？**
A: 需要 Super_Admin 权限，且操作 IP 必须在白名单内。进入导出页面选择「原始数据导出」。
